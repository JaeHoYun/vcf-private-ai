# 07 — 설계 리뷰 체크리스트와 검증 관문

[← 목차로](../README.md)

설계를 끝내기 전에, **모든 결정이 내려졌고 근거가 기록됐는지**를 점검합니다. 이 문서는 결정별 체크리스트와 단계별 검증 관문, 그리고 추적성 점검을 제공합니다.

> 본 문서의 항목은 작성 시점(2026-06) VCF 9.1 / PAIF 9.1 / PAIS 2.1 기준이며, 적용 전 공식 문서로 재확인하시기 바랍니다.

---

## 7.1 결정 리뷰 체크리스트

각 결정(D1–D12)에 대해 세 가지를 확인합니다. (1) 경로를 골랐는가, (2) [06.3 설계 결정 기록](06-decision-forks.md)으로 근거를 남겼는가, (3) 관련 형제 가이드(①–⑥)를 확인했는가.

| 결정 | 경로 선택 | 결정 기록 | 형제 가이드 확인 |
|------|:--------:|:--------:|:----------------|
| D1 토폴로지 | ☐ | ☐ | ① 구축 시나리오 |
| D2 GPU 공유 | ☐ | ☐ | ⑥ GPU 사이징 |
| D3 서빙 배치 | ☐ | ☐ | ③ 서빙 |
| D4 서빙 방식 | ☐ | ☐ | ③ 서빙 · ④ RAG |
| D5 네트워킹 | ☐ | ☐ | ⑤ 네트워크 보안 |
| D6 로드밸런서 | ☐ | ☐ | ③ 엔드포인트 |
| D7 스토리지 | ☐ | ☐ | ⑥ 스토리지 사이징 |
| D8 VectorDB | ☐ | ☐ | ② VectorDB |
| D9 가용성·재해복구(DR) | ☐ | ☐ | ① 백업·복구 |
| D10 테넌시 | ☐ | ☐ | ⑤ 격리 |
| D11 Identity | ☐ | ☐ | ⑤ 접근 통제 |
| D12 에어갭 | ☐ | ☐ | ⑤ 공급망 |

## 7.2 단계별 검증 관문

각 관문은 다음 단계로 넘어가기 전에 충족해야 할 조건입니다.

- **Gate 0 — 요구사항** : [결정 요인 시트](../worksheet/01-driver-sheet.md)가 수치·등급으로 채워졌는가. 서비스 수준 목표(SLO)가 측정 가능한가.
- **Gate 1 — 토폴로지·컴퓨트** : D1–D4 결정·기록 완료. GPU 하드웨어가 선택한 공유 방식(MIG 등)을 지원하는가. 최소 호스트 요건 충족.
- **Gate 2 — 네트워크·스토리지** : D5–D8 결정·기록 완료. VPC 셀프서비스가 목표면 오버레이 전제 충족. 프린시플 스토리지 선택이 VCF 9.1 지원 범위인가(iSCSI 프린시플 불가).
- **Gate 3 — 보안·테넌시** : D10–D12 결정·기록 완료. 에어갭 등급이 규제 요건과 일치. Identity 페더레이션·격리 모델이 감사 요건 충족.
- **Gate 4 — 가용성·운영 준비** : D9 결정·기록 완료. 패스스루 GPU의 HA 제약을 앱 레벨 replica로 보완. 백업·복구·수명주기 관리(LCM) 운영 계획(① Day-2) 연결.

## 7.3 추적성 점검

설계가 끝나면 다음 체인이 끊기지 않는지 확인합니다.

```
결정 요인(01)  →  설계 결정(06 카탈로그)  →  설계 결정 기록(06.3)  →  블루프린트 조정(02)
```

- 모든 결정 기록이 하나 이상의 결정 요인으로 거슬러 올라가는가(근거 없는 결정 없음).
- 블루프린트에서 바꾼 결정이 모두 설계 결정 기록에 남았는가.
- 형제 가이드로 위임한 항목(보안 상세·사이징 수치)이 해당 가이드에서 검증됐는가.

> 결정·근거·검증의 추적 체인이 이어져야 나중에 "왜 이렇게 설계했나"에 답할 수 있습니다.
