# 02 설계 결정 기록 (D1–D12)

> 이 워크시트는 [문서 03–06](../docs/06-decision-forks.md)과 함께 씁니다.
> 채우면 **설계 결정 기록**이 되고, [07 리뷰 관문](../docs/07-design-review.md)과 구축 인계의 출발 자료가 됩니다.

- **누가** — 설계 아키텍트가 이해관계자와 합의해 기록합니다.
- **언제** — 본문의 각 설계 결정을 읽고 정할 때마다, [01 결정 요인 시트](01-driver-sheet.md)를 근거로 채웁니다.

## 파트 A — 결정 요약표

각 결정의 **선택지에서 고른 것만 남기고** 나머지는 지웁니다. 근거 칸에는 그 선택을 끌어낸 결정 요인을 적습니다. 상태: `☐ 미정` · `☑ 결정` · `⚠ 확인 필요`.

| D# | 결정 | 선택지 (고른 것만 남기기) | 권고 | 근거 (연결 결정 요인) | 상태 |
|----|------|--------------------------|------|---------------------|:---:|
| D1 | VCF 토폴로지 | 표준 / 통합 | 표준 |  | ☐ |
| D2 | GPU 공유 | MIG / 타임슬라이싱 / 패스스루 | 워크로드별 |  | ☐ |
| D3 | 서빙 배치 | VKS / DLVM | VKS(프로덕션) |  | ☐ |
| D4 | 서빙 방식 | PAIS Model Runtime / NIM / 자가 vLLM | PAIS(표준) |  | ☐ |
| D5 | 네트워킹 | NSX 오버레이·VPC / 물리 VLAN | 요구별 |  | ☐ |
| D6 | 로드밸런서 | AVI / 내장 L4 / 서드파티 | AVI(프로덕션) |  | ☐ |
| D7 | 스토리지 | vSAN / 외장(NFS·FC·vVol) | 자산별 |  | ☐ |
| D8 | VectorDB | DSM pgvector / 외부 전용 | pgvector(표준) |  | ☐ |
| D9 | 가용성·DR | 단일 / stretched / 멀티사이트 | 연속성 요구별 |  | ☐ |
| D10 | 테넌시 격리 | soft / hard | 신뢰 경계별 |  | ☐ |
| D11 | Identity | 내장 / 외부 IdP 페더레이션 | 기업 표준별 |  | ☐ |

## 파트 B — 상세 결정 기록

되돌리기 어렵거나 트레이드오프가 큰 결정(특히 **D1 토폴로지 · D2 GPU 공유 · D5 네트워킹 · D7 스토리지 · D12 에어갭**)은 아래 양식으로 한 건씩 상세히 남깁니다. 결정 항목마다 아래 양식을 하나씩 복사해 채우세요. (양식 설명은 [06.3 설계 결정 기록 템플릿](../docs/06-decision-forks.md) 참조.)

```
결정 기록 NNN: [예: D7 스토리지]
- 날짜 / 작성자:
- 맥락: 어떤 결정 요인·제약 때문에 결정이 필요했나
- 선택: 채택한 경로 (예: 외장 NFS)
- 대안·기각 사유: 검토했으나 버린 경로와 이유 (예: vSAN — 신규 HCI 도입 부담)
- 결과·트레이드오프: 이 선택으로 감수하는 것 (예: 기존 SAN팀 운영 의존)
- 재검토 트리거: 무엇이 바뀌면 다시 본다 (예: SAN 노후 교체 시점)
```

```
결정 기록 NNN: [          ]
- 날짜 / 작성자:
- 맥락:
- 선택:
- 대안·기각 사유:
- 결과·트레이드오프:
- 재검토 트리거:
```
