## 요약

Private AI 사이징은 "어떤 AI 서비스를, 어느 규모로, 얼마에 운영할 것인가"를 GPU·서버·라이선스·운영비로 환산하는 작업입니다. **과소 산정은 장애·지연을, 과대 산정은 비용 낭비를** 부릅니다. 정답은 한 번에 나오지 않으며, **예산 추정 → PoC 실측 → 확정**의 순서로 좁혀 갑니다.

## 결정해야 할 것 (4가지)

| 결정 | 핵심 질문 | 영향 |
|---|---|---|
| 모델 규모 | 작은(7–13B) vs 중·대형(30–70B+)? | 이후 GPU·노드·비용 전체를 좌우. RAG는 작게 시작 가능 |
| 배포 형태 | 온프레미스 vs 퍼블릭 클라우드? | 비용 구조(고정비 vs 사용량)·데이터 주권 |
| 단계 | PoC부터 vs 바로 프로덕션? | 추정 위험. PoC 없이 확정 사이징은 불가에 가까움 |
| 데이터 주권 | 사내 통제(에어갭) 필요? | 온프레미스·격리·반입(Artifact Mirroring Tool) 요건 |

---

## 온프레미스 vs 퍼블릭 — 손익분기는 "사용률의 함수"

특정 % 절감을 단정할 수 없습니다. 우열은 워크로드·사용률·기간에 달려 있습니다([07.6](07-tco-cost-model.md#76-퍼블릭-gpu-클라우드-vs-온프레미스-paif-비교-프레임)).

- **온프레미스는 사용률이 높을수록 유리**합니다(24/7 고부하 추론·학습 등 꾸준한 수요).
- **퍼블릭은 간헐적·버스트성 수요나 초기 불확실성이 클 때 유리**합니다.
- 따라서 결정 전 **예상 가동률**을 먼저 추정하는 것이 가장 중요합니다.

> **벤더 절감 수치 주의**: "서버 최대 40%·운영비 최대 46% 절감" 같은 발표 수치는 best-case("up to")이며 자사 환경에 그대로 적용되지 않습니다. 의사결정에는 **자사 실측치**를 사용하세요([07.6](07-tco-cost-model.md#76-퍼블릭-gpu-클라우드-vs-온프레미스-paif-비교-프레임)).

## 비용 구조 (단가는 견적으로 확정)

| 대분류 | 내용 | 비고 |
|---|---|---|
| 소프트웨어 | VCF 코어당 구독(PAIF 포함) + NVAIE(GPU당, 별도) + DSM | PAIF는 VCF에 포함 → 이중 계상 금지 |
| 하드웨어 | GPU + 서버 + 스토리지 + 네트워크 | GPU 진부화 빨라 상각연수 보수적 |
| 운영비 | 인력 + 전력(GPU 전력×PUE) + 상면 + 유지보수 | 전력은 GPU TCO의 큰 변수 |
| 도입 | PoC·이행·교육 | 일회성 |

> 본 시리즈는 실제 단가를 제시하지 않습니다. 빈 단가는 [부록 A3 견적 요청 체크리스트](../appendix/A3-rfq-quote-checklist.md)로 벤더에서 확정합니다.

## 규모 감각 (레퍼런스 예시)

사내 임직원 5,000명 한국어 문서 RAG 챗봇은 **GPU 약 3장 / 9 노드 / 가용 스토리지 약 0.6TB** 규모로 추정됩니다([08](08-reference-scenario.md), 모두 1차 가정치·PoC로 확정). 시사점:

- **RAG의 GPU 수요는 의외로 작습니다**(지식이 모델 밖 벡터DB에 있어서).
- **소규모일수록 플랫폼 최소요건·고가용성 구성이 비용을 지배**합니다.
