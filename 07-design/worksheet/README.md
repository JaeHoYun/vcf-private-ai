# 채워넣기 워크시트 (통합 설계)

> 기반 버전은 [상위 README](../README.md)와 [① 버전 기준 문서](../../01-infra/README.md#기반-버전-source-of-truth)를 참조하세요.

⑦ 통합 설계 가이드 본문이 "각 결정을 **어떻게** 내리나"를 설명한다면, 이 워크시트는 **설계 결정을 직접 적어 한 장의 설계 결정 기록으로 남기는** 채워넣기 양식입니다. 계산용 스프레드시트(xlsx)가 아니라, 항목을 따라 적어 넣는 마크다운입니다(수치 산정은 사이징·비용 편 ⑥의 계산 워크북을 씁니다).

## 워크시트 구성

| 워크시트 | 무엇을 채우나 | 주 작성자 | 언제 | 채운 뒤 |
|---|---|---|---|---|
| [01 요구사항·제약 시트](01-driver-sheet.md) | 요구사항·제약을 수치·등급으로 수집 | 아키텍트 + 업무·보안·예산 담당 | 설계 착수 전(요구사항 워크숍) | 설계 결정의 입력, 제안서 요구사항 절 |
| [02 설계 결정 기록](02-design-decision-record.md) | D1–D12 선택 경로와 근거 | 아키텍트(이해관계자 합의) | 설계 결정을 내릴 때 | 설계 결정 기록, 리뷰(07)·구축 인계 자료 |

## 사용법

1. **01 요구사항·제약 시트**를 먼저 채웁니다(요구사항 워크숍). 모르는 칸은 비우고 "확인 필요"로 표시합니다.
2. 본문 **03–05 설계 결정**을 읽으며 **02**에 결정을 적습니다. 각 결정의 근거는 [06 카탈로그](../docs/06-decision-forks.md)의 권고 트리거와 01에서 채운 요구사항·제약입니다.
3. 02를 [02 블루프린트](../docs/02-reference-blueprints.md)와 대조해, 어긋나는 결정만 조정합니다.
4. [07 검증 관문](../docs/07-design-review.md)로 누락·근거 없는 결정을 점검합니다.

## 이 워크시트가 쓸모 있으려면 (작성 원칙)

- **다 채우지 않아도 됩니다** — 01은 각 항목에 `필수 / 권장 / 선택` 우선순위를 표시했습니다. 필수부터 채우세요.
- **출처·측정을 적습니다** — 각 요구사항·제약 항목에 "어디서 얻고 어떻게 측정하는지" 칸이 있습니다. 측정 못 하는 값은 결정을 끌 수 없습니다.
- **다음 단계로 이어집니다** — 채운 값이 어느 결정(D#)을 미는지 연결돼 있어, 입력 → 결정 → 기록이 한 줄로 추적됩니다.

## 중요(반드시 읽기)

- 이 워크시트는 **채워넣기 양식**입니다. 자동 계산은 없으며, 항목별로 값을 적고 판단을 기록하는 용도입니다.
- GPU·노드·용량·TCO 등 **수치 산정이 필요하면** 사이징·비용 편(⑥)의 계산 워크북을 사용하고, 결과만 여기에 옮겨 적습니다.
- 모든 항목은 작성 시점 기준이며 환경·릴리스별로 달라집니다. 적용 전 [상위 README](../README.md)와 공식 문서로 재확인하세요.
