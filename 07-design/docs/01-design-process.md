# 01 — 설계 프로세스와 요구사항 수집

[← 목차로](../README.md)

좋은 Private AI 설계는 제품 기능에서 시작하지 않습니다. **요구사항과 제약(결정 요인)** 에서 시작해, 그 결정 요인이 어떤 설계 결정을 어느 방향으로 좌우하는지를 따라갑니다. 이 문서는 결정 요인을 수집하는 방법과 결정의 순서를 정합니다.

> 본 문서의 수치·동작은 작성 시점(2026-06) VCF 9.1 / PAIF 9.1 / PAIS 2.1 기준이며, 적용 전 공식 문서로 재확인하시기 바랍니다.

---

## 1.1 설계 프로세스 개요

```
요구사항·제약 수집(결정 요인)  →  결정 요인이 미는 설계 결정 식별
   →  설계 결정별 경로 선택(06 카탈로그)  →  가장 가까운 블루프린트 선택·조정(02)
   →  설계 결정 기록으로 선택·근거 남기기(06.3)  →  설계 리뷰·검증 관문(07)
```

핵심은 **결정 요인이 결정을 끌고 간다**는 점입니다. 설계 결정 카탈로그([06](06-decision-forks.md))는 결정의 관점에서, 이 문서는 요구사항의 관점에서 같은 결정 요인-설계 결정 대응 관계를 다룹니다.

## 1.2 결정 요인 수집

아래 여덟 가지를 먼저 채웁니다. 각 결정 요인은 어떻게 측정·확인하는지와, 그것이 강하게 좌우하는 결정(D1~D12)을 함께 적었습니다.

| 결정 요인 | 무엇을 확인하나 | 강하게 미는 결정 |
|---------|----------------|-----------------|
| 워크로드 프로파일 | 추론/학습/RAG/에이전트 비중, 모델 크기, 동시 사용자·요청 | D2 GPU 공유 · D3 서빙 배치 · D4 서빙 방식 |
| 성능 SLO | 첫 토큰 지연(TTFT)·E2E 지연 목표, 처리량(토큰/초) | D2 GPU 공유 · D4 서빙 방식 · D9 가용성 |
| 규제·데이터 주권 | 에어갭 의무, 격리 등급, 감사·접근 통제 요건 | D12 에어갭 · D10 테넌시 · D11 Identity |
| 예산 | CapEx/OpEx 한도, 최소 호스트 수, 라이선스 예산 | D1 토폴로지 · D2 GPU 공유 |
| 기존 인프라 자산 | 물리망(VLAN)·SAN/NAS 투자, GPU 보유, NVAIE 라이선스 | D5 네트워킹 · D6 LB · D7 스토리지 |
| 운영 스킬셋 | NSX·쿠버네티스·스토리지 운영 역량 | D3 서빙 배치 · D5 네트워킹 · D8 VectorDB |
| 멀티테넌시 요건 | 테넌트 수·신뢰 경계, 셀프서비스 필요 | D10 테넌시 · D5 네트워킹 |
| 가용성·연속성 | 허용 RPO/RTO, 사이트 수, 무중단 요구 | D9 가용성·DR · D2 GPU 공유(HA 제약) |

> 결정 요인은 "있으면 좋은 것"이 아니라 **수치·등급으로** 적어야 결정을 끌 수 있습니다. 예: "지연 빠르게"가 아니라 "TTFT P95 1.5초 이하".

## 1.3 설계 결정 순서

의존성이 있는 순서로 결정합니다. 앞 결정이 뒤 결정의 선택지를 좁히기 때문입니다.

1. **D1 토폴로지** — 표준이 기본(통합은 비권고). 설치 시 확정되므로 가장 먼저.
2. **D2 GPU 공유** — 하드웨어 선정(MIG 지원 여부)과 묶임.
3. **D3·D4 서빙(배치·방식)** — 워크로드 프로파일을 직접 반영.
4. **D5·D6 네트워크·LB** — 셀프서비스·VPC 여부가 LB 선택에 영향.
5. **D7·D8 스토리지·VectorDB** — 기존 자산과 RAG 규모.
6. **D9 가용성·DR** — 사이트·링크 요건, GPU HA 제약 반영.
7. **D10–D12 테넌시·Identity·에어갭** — 규제·조직 경계.

각 결정은 [06 카탈로그](06-decision-forks.md)에서 경로를 고르고 [06.3 설계 결정 기록 템플릿](06-decision-forks.md)으로 근거를 남깁니다.

## 1.4 이 단계의 산출물

- 채워진 **[결정 요인 시트](../worksheet/01-driver-sheet.md)**(수치·등급)
- 가장 가까운 **블루프린트 선택**([02](02-reference-blueprints.md))과 조정 메모
- 결정별 **[설계 결정 기록](../worksheet/02-design-decision-record.md)**([06.3](06-decision-forks.md))
