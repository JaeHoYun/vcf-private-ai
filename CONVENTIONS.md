# 가이드 넘버링·구조 규약 (시리즈 공통)

VCF Private AI 가이드 시리즈(①–⑦)의 모든 편(서브폴더)은 아래 규약을 따른다. 에이전트 서비스 가이드(`vcf-private-ai-agents`)는 시리즈 번호 챕터가 아니라 플랫폼 위에 올리는 별도 최상위 가이드이며, 규약상 다른 편을 가리킬 때는 '에이전트 가이드'로 표기한다.
문서 번호 체계, 파일 배치, 헤딩 넘버링을 일관화해 편 간 상호참조와 유지보수를 단순하게 유지하려는 규약이다.
기준 문서는 ⑥ 사이징·용량·비용의 구조이며, 신규 문서·기존 문서 모두 본 규약에 맞춘다.

## 1. 시리즈 레벨

- 각 가이드는 시리즈 번호 ①–⑦로 식별한다. 식별의 기준 문서는 허브 README의 "가이드 목록" 표다.
- 가이드 식별 번호(①–⑦)와 문서 내부 넘버링(아래)은 분리한다.
- 본문 헤딩에 ①②… 같은 원숫자 기호를 섞지 않는다. 다른 편을 가리킬 때만 사용한다(§6).

## 2. 파일·디렉터리

| 구분 | 경로·이름 | 예시 |
|------|-----------|------|
| 본문 문서 | `docs/NN-slug.md` (NN = 2자리 0패딩, `01`부터) | `docs/03-retrieval-context.md` |
| 전문/오리엔테이션 | `docs/00-*.md` | `docs/00-orientation.md` |
| 별도 독자트랙(임원 요약 등) | `docs/E0-*.md` (E = Executive) | `docs/E0-executive-brief.md` |
| 부록 | `appendix/AN-slug.md` (A1, A2, …) | `appendix/A1-first-order-reference.md` |

- 본문 흐름상 일반 장(章)에 해당하는 문서는 부록으로 빼지 않고 `docs/NN`에 둔다.
- 진짜 부록(참조표·체크리스트·비교자료 등 본문 흐름 밖의 자료)만 `appendix/`로 모은다.

## 3. 문서 제목 (H1)

- 형식: `# NN — 제목` (구분자는 em대시 " — ", 마침표나 "Part" 접두 금지)
- 부록: `# A1 — 제목`
- 전문/오리엔테이션: `# 00 — 제목`, 별도 독자트랙: `# E0 — 제목`

## 4. 본문 헤딩

- H1은 문서 자체(레벨 1)다. 별도의 `## 1` 헤딩은 두지 않는다.
- H2 = `## N.M` (N = 해당 문서 번호, 0패딩 없이) — 항상 번호를 붙인다.
- H3 = `### N.M.K` — 구조상 의미 단위면 번호를 붙인다.
- H4 이하는 번호를 붙이지 않는다. `#### HNSW`, `#### Week 1`처럼 서술형 라벨로 둔다.

예) 문서 `01` 내부: `## 1.1` → `### 1.6.1` → 그 아래 `#### 세부 항목`(무번호)

- 부록은 선두를 A로 둔다: `## A1.1`, `### A1.1.1`
- 번호는 최대 3단계(N.M.K)까지만 둔다. 더 깊은 구조가 필요하면 절을 분리한다.

## 5. 비번호 섹션

- 문서 말미의 "참고 출처 / 요약 / 면책"류 메타 섹션에는 번호를 붙이지 않는다.

## 6. 상호참조

- 같은 가이드 안: `§N.M` 또는 문서 번호 `NN`으로 참조한다.
- 다른 가이드: "시리즈 ④의 §2.3"처럼 편 기호로 명시한다. 편을 가로지르는 전역 번호는 사용하지 않는다.
