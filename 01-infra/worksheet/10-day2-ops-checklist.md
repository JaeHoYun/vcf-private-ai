# 워크시트 — 10 Day-2 운영 점검·기록

> [문서 10 — Day-2 운영](../docs/10-operations.md)과 함께 쓰는 운영 기록·점검 템플릿입니다.
> 본문이 "무엇을 어떻게 하나"를 설명한다면, 이 워크시트는 **실제 점검일·담당·실측값을 채워 넣어 기록**하는 용도입니다.
> 환경에 맞게 행을 더하거나 빼서 쓰세요. 빈칸은 운영 중 채워 갑니다.

- **누가** — 플랫폼 운영자·SRE가 점검할 때마다 담당 칸에 실명으로 기록합니다. 업그레이드는 변경관리 담당과 함께 진행합니다.
- **언제** — 구축 이후 상시(일·주·월·분기 리듬), 그리고 업그레이드·복구 훈련마다 사본을 만들어 채웁니다.

## 섹션 4 — SLO 기준선 기록표

§10.4.1 기반. SLI별로 **기준선(baseline)을 실측**한 뒤 목표 SLO를 설정합니다. 아래 목표값은 형식 예시이며 실제 목표가 아닙니다.
**측정 방법·출처** 칸에는 각 지표를 어디서·어떻게 읽는지(VCF Operations 지표 화면, [문서 06 §6.8 관측성](../docs/06-production.md) 대시보드, 관측 백엔드 쿼리)를 적어, 누가 봐도 같은 값을 다시 잴 수 있게 합니다.

| SLI | 지표(예) | 측정 방법·출처(예) | baseline 실측값 | 목표 SLO | 측정일 | 비고 |
|-----|---------|------------------|----------------|---------|--------|------|
| 가용성 | Endpoint 요청 성공률(2xx) | VCF Operations Endpoint 지표 · 관측 백엔드에서 2xx/전체 비율 |  |  |  |  |
| 지연 | TTFT, E2E P95 지연 | VCF Operations AI 지표(TTFT·E2E) · 백엔드 P95 패널 |  |  |  |  |
| 오류율 | Endpoint 오류율 | VCF Operations Endpoint 지표 · 5xx/전체 비율 |  |  |  |  |
| GPU 건전성 | GPU 사용률·메모리 압력 | VCF Operations GPU 지표 · 임계 초과 시간 비율 |  |  |  |  |
