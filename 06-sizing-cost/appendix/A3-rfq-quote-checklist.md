# 부록 A3 — 견적 요청(RFQ) 체크리스트

> 기반 버전은 [README 버전 기준 문서](../README.md#기반-버전-source-of-truth)를 참조하세요.
> 시리즈 인덱스: [vcf-private-ai-series](../../README.md)

본 시리즈는 실제 단가를 제시하지 않습니다([07](../docs/07-tco-cost-model.md)의 단가 칸은 모두 견적 입력란). 이 부록은 그 빈칸을 채우려면 **누구에게 무엇을 물어야 하는가**를 정리한 견적 요청 체크리스트입니다. 단가 대신 **무엇을 채워야 하는지 안내하는 실무 도구**(질문 목록·산정 규칙·기록 양식)를 담습니다.

> 입력으로 쓸 사이징 결과(코어 수·GPU 수·노드 수·용량)는 [08 레퍼런스 시나리오](../docs/08-reference-scenario.md)·[04.6](../docs/04-vks-cluster-sizing.md#46-인프라-고려사항과-사이징-워크시트)·[05.5](../docs/05-storage-network-sizing.md#55-사이징-워크시트--규모에서-용량으로)에서 가져옵니다.

---

## A3.1 소프트웨어 라이선스 견적 (Broadcom / NVIDIA)

라이선스는 TCO에서 오해가 가장 많은 항목입니다([07.2](../docs/07-tco-cost-model.md#72-소프트웨어-라이선스구독-비용)). 다음을 명확히 질의하세요.

### VCF 코어 구독 (Broadcom)

| 질문 항목 | 왜 필요한가 |
|---|---|
| 물리 코어당 단가와 약정 기간(연 단위) | 연 환산 TCO의 기준 |
| **코어 최소수량(core minimum) 규칙과 실제 값** | 코어 적은 CPU도 최소수량이 적용돼 "장부상 코어"가 늘 수 있음 → 사이징 물리 코어를 그대로 쓰면 과소 산정 |
| 솔루션/에디션 구성과 PAIF 포함 범위 | PAIF는 VCF 구독에 포함(별도 제품 아님) — 이중 계상 방지 |
| DSM(Data Services Manager) 사용 권한(entitlement) 조건 | 벡터DB(pgvector) 프로덕션 사용 조건 |
| 스토리지 구성에 따른 코어 영향 | vSAN vs 외부 스토리지가 노드 수·코어 수에 영향([07.4](../docs/07-tco-cost-model.md#74-서버스토리지네트워크gpu-외-인프라)) |

### NVAIE (NVIDIA AI Enterprise, NVIDIA가 별도로 판매)

| 질문 항목 | 왜 필요한가 |
|---|---|
| **GPU당(per-GPU) 단가** | 설치된 모든 물리 GPU에 라이선스 필요(vGPU 분할 무관) |
| 에디션과 기능 범위 | 기능·지원 차이 |
| 구독 vs 소비형 vs 영구 / 지원 기간 | 연 환산이 크게 달라짐(영구는 별도 지원 서비스 필요) |
| DLS/CLS 라이선스 서버 요건 | 라이선스 배포·집계 인프라 |

## A3.2 하드웨어 견적 (GPU·서버·스토리지·네트워크)

사이징([04](../docs/04-vks-cluster-sizing.md)·[05](../docs/05-storage-network-sizing.md))의 수량을 그대로 견적 입력으로 씁니다.

| 항목 | 질문할 사양 | 사이징 입력 |
|---|---|---|
| GPU | 모델·메모리(HBM)·수량·단가·납기 | 총 GPU 수([02](../docs/02-gpu-sizing.md)) |
| GPU 서버(노드) | 폼팩터·호스트당 GPU 슬롯·CPU·RAM·단가 | 노드 수·노드 사양([04.3](../docs/04-vks-cluster-sizing.md#43-노드-사양-산정-노드-크기-vs-노드-수)) |
| 고속 패브릭 NIC | 포트 수·속도 | 노드당 포트([07.3](../docs/07-tco-cost-model.md#73-하드웨어-비용capex와-감가)) |
| GPU 간 인터커넥트 | NVLink/스위치 구성 | 토폴로지([04.3](../docs/04-vks-cluster-sizing.md#43-노드-사양-산정-노드-크기-vs-노드-수)) |
| 스토리지 | 용량·미디어(NVMe)·단가 | 필요 가용 용량([05.5](../docs/05-storage-network-sizing.md#55-사이징-워크시트--규모에서-용량으로)) |
| 네트워크 스위치 | 관리/워크로드 토폴로지·수량 | [05.4](../docs/05-storage-network-sizing.md#54-네트워크-용량--추론반입내부-통신) |

> GPU·드라이버·프로파일이 PAIF 지원 매트릭스와 서버 벤더의 BCG(BIOS·펌웨어 호환성)/HCL(하드웨어 호환성 목록, Hardware Compatibility List)에 부합하는지 함께 확인하세요([02.8](../docs/02-gpu-sizing.md#28-gpu-모델-선택-가이드-hbm-용량세대)).

## A3.3 시설·운영·도입 견적

하드웨어·라이선스만 보면 누락되기 쉬운 항목입니다([07.5](../docs/07-tco-cost-model.md#75-운영비opex와-도입마이그레이션)).

| 항목 | 질문할 데이터 |
|---|---|
| 전력 | 전력 단가, 사이트 PUE, 노드·GPU 소비전력(kW) |
| 상면/냉각 | 랙 수·열밀도·상면 단가 |
| 유지보수/지원 | 하드웨어·소프트웨어 계약 단가(구독 포함 여부) |
| 도입/마이그레이션 | PoC·이행·교육·컨설팅(일회성) |

## A3.4 견적 추적 기록 양식

채운 단가는 **출처를 기록**해 추적 가능성을 유지합니다([07.8](../docs/07-tco-cost-model.md#78-검증실측-방법)). 아래 양식을 복사해 쓰세요.

```
| 항목 | 벤더 | 단가 | 통화 | 견적번호 | 일자 | 유효기간 | 비고 |
| VCF 코어/년       |  |  |  |  |  |  | 최소수량 반영 여부 |
| NVAIE GPU/년      |  |  |  |  |  |  | 에디션·기간 |
| DSM               |  |  |  |  |  |  | 사용 권한(entitlement) 조건 |
| GPU               |  |  |  |  |  |  | 모델·수량 |
| GPU 서버          |  |  |  |  |  |  | 폼팩터 |
| 스토리지          |  |  |  |  |  |  | 용량·미디어 |
| 네트워크          |  |  |  |  |  |  |  |
| 전력(실효)        |  |  |  |  |  |  | PUE 반영 |
| 유지보수          |  |  |  |  |  |  |  |
| 도입/마이그레이션 |  |  |  |  |  |  | 일회성 |
```

## A3.5 산정 주의 (견적을 TCO에 넣기 전)

- **코어 최소수량 반영**: 사이징 물리 코어가 아니라 최소수량 규칙 적용 후 코어로 계산([07.2](../docs/07-tco-cost-model.md#72-소프트웨어-라이선스구독-비용)).
- **NVAIE는 설치 GPU 전수**: vGPU 분할과 무관하게 물리 GPU 수 기준.
- **이중 계상 금지**: PAIF는 VCF 구독에 포함.
- **상각연수 일관성**: GPU·서버 상각연수를 분석 기간([07.1](../docs/07-tco-cost-model.md#71-tco-범위와-기간-설정))과 맞춤.
- **분기별 재산정**: 사용률·요금·환율 변동을 반영([07.8](../docs/07-tco-cost-model.md#78-검증실측-방법)).

> 단가는 본 문서가 제시하지 않습니다. 모든 빈칸은 공식 견적·구독 조건으로 채우고, 채운 출처(견적번호·일자)를 남기세요.
