# 07 — 프로덕션 운영

[← 목차로](../README.md)

PoC를 전사 서비스로 올리는 단계입니다. 세부 인프라 운영은 ①(06 프로덕션)과 ②(Day-2 런북)이 기준 문서이며, 여기서는 **RAG 시스템 관점의 운영 결정**을 모읍니다.

## 7.1 스케일링

RAG는 세 축에서 독립적으로 부하가 옵니다.

| 구성요소 | 부하 성격 | 스케일 전략 |
|----------|-----------|-------------|
| 완성 모델(LLM) | GPU 병목(GPU-bound), 동시 추론 | Model Runtime 레플리카·GPU 증설(①③) |
| 임베딩 모델 | 쿼리 임베딩 + 배치 인덱싱 | 인덱싱은 배치 분리, 쿼리용 별도 용량 |
| pgvector | 검색 QPS(초당 쿼리 수), 인덱스 메모리 | 인덱스 튜닝·읽기 확장·리소스 분리(②) |

> 인덱싱(배치, 무거움)과 쿼리(실시간, 가벼움)의 임베딩 트래픽을 **분리**하세요. 대량 재인덱싱이 사용자 질의 지연을 끌어올리지 않게 합니다.

## 7.2 멀티테넌트 / 격리

여러 부서·조직이 한 플랫폼을 공유할 때:

- **VCF Automation UI 네임스페이스**: 9.1에서 조직 관리자가 네임스페이스 단위로 PAIS를 활성화·관리합니다. 테넌트 경계를 네임스페이스로 잡는 것이 출발점(①).
- **Knowledge Base 분리**: 테넌트별 KB/인덱스를 분리하거나, 공유 인덱스에 **메타데이터 + 검색 필터**로 논리 격리(03·05). 민감도가 높으면 물리 분리.
- **데이터 누수 방지**: 검색 필터가 곧 보안 경계입니다. 필터 누락은 타 부서 문서 노출로 직결되니 회귀 테스트에 **권한 격리 케이스**를 포함(06).

## 7.3 캐시

- **임베딩 캐시**: 동일/유사 질문의 질문 임베딩·검색 결과 캐시로 지연·GPU 절감.
- **응답 캐시**: 자주 묻는 질문(FAQ성)은 답변 캐시. 단 **문서 갱신 시 무효화** 정책 필수(낡은 답 방지).
- **무효화 연동**: 02의 재인덱싱과 캐시 무효화를 같은 이벤트로 묶으세요.

## 7.4 사이징 / 비용

- **GPU 사이징**: 완성 모델 크기·동시 사용자·목표 지연으로 산정. 리랭킹을 켜면 GPU 수요가 추가됩니다(03). 구체 사이징은 ① 참조.
- **공배치(pgvector + GPU)**: pgvector를 GPU 컴퓨트와 같은 vSAN 인프라에 두면 RAG·추론 지연이 줄어듭니다(②).
- **비용 조절 수단**: 청크 크기·top-k·리랭킹 후보 수는 모두 토큰·GPU 비용에 직결됩니다. 06의 품질과 비용을 함께 최적화하세요.

## 7.5 에어갭(폐쇄망)

방산·금융·공공 등 폐쇄망 RAG:

- **Artifact Mirroring Tool**: PAIS 2.1의 에어갭 구동 도구로, GPU 기반 Model Endpoint와 에이전트를 포함한 풀 Private AI 기능을 폐쇄망에서 운용합니다(①). 모델·이미지·아티팩트를 미러링으로 반입.
- **RAG 관점**: 임베딩·완성·리랭커 모델과 인덱싱 파이프라인이 모두 폐쇄망 안에서 자족해야 합니다. 외부 임베딩 API 의존이 없는지 점검.
- 산업별 시나리오는 ① 07 참조.

## 7.6 신뢰성 / Day-2

- **HA/DR(고가용성·재해복구)**: pgvector(②)·Model Runtime(①)의 고가용성·백업 정책을 따름. 인덱스는 재생성 가능하지만 **재인덱싱 시간**이 RTO(목표 복구 시간)에 들어가니 백업 전략을 결정하세요.
- **모델 라이프사이클**: 모델 교체 시 임베딩 모델이 바뀌면 **전면 재인덱싱**(02). 완성 모델만 바뀌면 인덱스는 유지, 06 회귀만.
- **운영 대시보드**: 6.5(관측성)의 지표를 상시 노출.

## 7.7 핵심 결정 요약

| 결정 | 기본 권장 |
|------|-----------|
| 임베딩 트래픽 | 인덱싱(배치)·쿼리(실시간) 분리 |
| 테넌트 격리 | 네임스페이스 + KB/필터, 민감 시 물리 분리 |
| 캐시 무효화 | 재인덱싱 이벤트와 연동 |
| 에어갭 | Artifact Mirroring Tool로 모델·아티팩트 자족 |
| 재인덱싱 | 임베딩 모델 교체 시 필수, RTO에 반영 |
