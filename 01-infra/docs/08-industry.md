# 08 — 한국 산업군 적용 시나리오

> 기반 버전은 [README 버전 기준 문서](../README.md#기반-버전-source-of-truth)를 참조하세요.
> 이 문서는 PAIF 9.1을 **한국 주요 산업군**에 적용하는 시나리오를 정리합니다. 일반 가이드(문서 01–06)를 산업 맥락으로 옮기는 다리 역할입니다.

PAIF는 **데이터를 외부로 반출하지 않고**(온프레미스/에어갭), **기존 VMware 자산 위에서**(VM+컨테이너+AI 단일 플랫폼) AI를 운영할 수 있다는 점이 한국 시장에서 핵심 강점입니다. 9.1의 **Artifact Mirroring Tool(에어갭)·Blackwell·MCP**가 이 강점을 직접 강화합니다.

> **고지:** 본 문서는 산업군 **일반**의 적용 패턴을 다룹니다. **특정 기업·고객사를 지칭하거나 그 도입 사실을 단언하지 않습니다.** 모든 시나리오는 가상의 일반 예시이며, 실제 적용은 각 조직의 환경·규제·데이터 실측을 전제로 검토하시기 바랍니다.

---

## 8.1 9.1 킬러 기능과 산업 매핑

| 9.1 기능 | 핵심 가치 | 특히 중요한 산업 |
|---------|----------|----------------|
| **Artifact Mirroring Tool (에어갭)** | 외부 반출 불가 환경에서 풀 AI 구동 | 방산, 금융, 공공, 일부 제조 |
| **Enhanced DirectPath I/O + Blackwell** | NVAIE(NVIDIA AI Enterprise) 없이 전용 GPU + vMotion, 대규모 학습 | 제조(비전/학습), 콘텐츠(생성형) |
| **MCP** | 사내 시스템(ERP·MES·ITSM·DB)과 표준 연동 | 전 산업 (특히 유통·제조 업무 자동화) |
| **CPU 추론(llama.cpp)** | GPU 없이 소규모 추론 비용 절감 | 전 산업 PoC/지점 단위 |
| **관측성·거버넌스** | 모델 품질·비용·감사 추적 | 금융, 공공 (규제 대응) |

---

## 8.2 제조업

라인 데이터·공정 데이터는 외부 반출이 어렵고, 현장 적용 가치가 큰 영역입니다.

| 시나리오 | 내용 | PAIF 연계 |
|---------|------|----------|
| **품질검사 비전 AI** | 라인 카메라 → 결함 분류 모델 | Blackwell/DirectPath로 학습·추론, 온프레미스 필수 |
| **예지보전(PdM)** | 설비 센서 시계열 → 이상감지/잔존수명(RUL, Remaining Useful Life) 예측 | VKS GPU 워크로드, vSAN 학습셋 |
| **공정 최적화 RAG** | 작업 표준서·트러블슈팅 매뉴얼 RAG 챗봇 | Knowledge Base + Agent |
| **MES/ERP 연동 에이전트** | "오늘 라인 불량률·자재 재고 알려줘" | **MCP**로 사내 DB/MES 연동 |
| **CAD/CAE 보조** | 설계 가이드 자연어 질의, BOM 자동화 | RAG + Tool-calling |

**도입 포인트:** 공정 데이터의 외부 반출 제약 → **온프레미스 PAIF + (필요 시 Artifact Mirroring Tool 에어갭)**. 대규모 비전 모델 학습은 **Blackwell + Enhanced DirectPath I/O(멀티호스트 학습)**.

---

## 8.3 방위산업

**외부 반출 절대 불가** — 9.1 Artifact Mirroring Tool의 가장 직접적인 수혜 산업입니다.

| 시나리오 | 내용 | PAIF 연계 |
|---------|------|----------|
| **에어갭 LLM/RAG** | 방산 기술문서 내부 RAG | **Artifact Mirroring Tool 폐쇄망 풀스택**, NIM/NeMo 오프라인 반입 |
| **기술문서 자동 생성** | 시방서·제안서 초안 | 내부 KB + Agent (모델·데이터 외부 반출 불가) |
| **시뮬레이션 보조** | 무기체계 시뮬레이션 결과 분석 | GPU 워크로드, 내부 데이터만 |

**도입 포인트:** 인터넷 연결 미러 호스트에서 **Artifact Mirroring Tool로 모델·컨테이너·드라이버를 미러링 후 오프라인 반입** → 내부 Harbor → GPU Model Endpoint·에이전트 구동. **MCP는 내부 시스템으로만 한정**하고, 외부 SaaS 연동은 차단합니다 ([문서 06 §6.9](06-production.md#69-에어갭air-gapped-환경--artifact-mirroring-tool-91-신규)).

## 8.4 유통/물류

개인정보·멤버십 데이터 보호와 그룹 단위 데이터 풀링이 관건입니다.

| 시나리오 | 내용 | PAIF 연계 |
|---------|------|----------|
| **수요예측/재고최적화** | 매장별 시계열 + 외부 변수 | 그룹 GPU 풀, VKS |
| **개인화 추천** | 멤버십 데이터 기반 | 온프레미스(개인정보보호법 준수) |
| **고객 응대 챗봇** | CRM 통합 RAG | KB + **MCP(CRM/주문 DB 연동)** |
| **상품 콘텐츠 자동 생성** | 상세페이지·카피라이팅 | Completion Endpoint (대량은 GPU, 소량은 CPU) |
| **업무 자동화 에이전트** | "재고 부족 상품 발주 티켓 생성" | **MCP(ServiceNow/ERP)** — 쓰기 작업 승인 게이트 |

**도입 포인트:** 개인정보 보호 → 온프레미스. 업무 자동화는 **MCP로 사내 ERP/CRM/ITSM 연동**하되, 발주·전송 등 **쓰기 작업은 거버넌스 승인 게이트** 필수 ([문서 05 §5.5](05-agents-mcp.md#55-거버넌스-가장-중요)).

---

## 8.5 콘텐츠/미디어

생성형 AI 활용도가 높고, 대규모 GPU와 저작권 거버넌스가 함께 요구됩니다.

| 시나리오 | 내용 | PAIF 연계 |
|---------|------|----------|
| **콘텐츠 모더레이션** | 자체 가이드라인 기반 AI 검수 | Agent + KB(가이드라인) |
| **자동 번역/더빙/자막** | 글로벌 OTT 다국어 워크플로우 | 다국어 모델, GPU 추론 |
| **스토리보드/이미지 생성 보조** | 생성형 AI | Blackwell GPU, **저작권 이슈 사전 정리** |
| **추천/큐레이션** | 시청 데이터 기반 자체 학습 | VKS GPU 학습 |

**도입 포인트:** 대규모 생성형 워크로드 → **Blackwell + DirectPath I/O**. 자체 데이터 학습으로 외부 모델 의존·데이터 유출 최소화. 저작권·라이선스는 도입 전 정리.

---

## 8.6 금융 (보조 영역)

규제·감사 대응과 내부 데이터 보호가 핵심입니다.

| 시나리오 | 내용 | PAIF 연계 |
|---------|------|----------|
| 내부 문서 RAG | 규정·약관·매뉴얼 | KB + Agent |
| 코드 어시스턴트 | 행 내부 코드 학습 금지 → 온프레미스 | Completion Endpoint |
| 콜센터 상담 보조 | 상담 스크립트·이력 RAG | KB + **MCP(상담 시스템)** |

**도입 포인트:** **관측성·거버넌스(감사 로깅, 모델 품질 추적, MCP 도구 감사)** 가 규제 대응에 직결됩니다. 폐쇄망 요건 시 Artifact Mirroring Tool.

---

## 8.7 단계적 도입 로드맵 (공통)

```
Phase 1: 단일 워크로드 PoC
  • RAG 챗봇 1개 (내부 문서) — 패턴 1(관리형 엔드투엔드)로 빠르게
  • CPU 추론(llama.cpp)으로 비용 최소 시작 가능
        ↓
Phase 2: 부서/계열사 확장
  • 네임스페이스 멀티테넌트, 리소스 쿼터
  • MCP로 사내 시스템 연동 (읽기 우선)
        ↓
Phase 3: 그룹 GPU 풀 + 프로덕션
  • Blackwell/DirectPath 대규모 워크로드
  • HA/DR, 관측성·거버넌스, (필요 시) Artifact Mirroring Tool 에어갭
```

**도입 비용 산정 시 필수 체크:**
- 지원 **NVAIE 별도 구매** 누락 금지 (vGPU 분할·NIM/NeMo 사용 시). Enhanced DirectPath I/O 전용 패스스루만 쓰면 NVAIE 불필요.
- 지원 **GPU 하드웨어 호환성 목록(BCG/HCL)** 사전 확인 — BCG: Broadcom Compatibility Guide / HCL: Hardware Compatibility List (Blackwell·ConnectX-7·BlueField-3 포함).
- 지원 **TCO 비교**(퍼블릭 GPU vs 온프레미스 PAIF)는 워크로드·사용률에 좌우 → 고객 실측 기반.
- 지원 **한국어 평가셋** 확보 (글로벌 벤치마크와 한국어 성능 차이 큼). 한국어 임베딩(bge-m3 등) 평가.
