# 채워넣기 워크시트 (구축·운영)

> 기반 버전은 [상위 README 버전 기준 문서](../README.md#기반-버전-source-of-truth)를 참조하세요.

문서 09 구축 시나리오와 문서 10 Day-2 운영을 **현장에서 채워 쓰는 워크시트**입니다. 구축을 시작하기 전 현황과 선결요건을 정리해 시나리오별 작업기술서(SoW)를 도출하고, 구축 이후에는 정기 점검·업그레이드·복구·SLO(서비스 수준 목표, Service Level Objective) 기록을 한곳에 남길 수 있습니다. 계산용 스프레드시트(xlsx)가 아니라, 항목을 따라 직접 적어 넣는 마크다운 양식입니다(수치 산정은 사이징·비용 편 ⑥의 계산 워크북을 씁니다).

## 워크시트 목록

| 워크시트 | 무엇을 채우나 | 주 작성자 | 언제 | 채운 뒤 |
|---|---|---|---|---|
| [09 구축 결정](09-deployment-decision.md) | 시나리오 결정·현황 파악·선결요건·SoW 정의 | 인프라·플랫폼 리드 + 아키텍트 | 구축 착수 전 | 착수 회의·제안서·SoW 출발 자료, 사이징은 ⑥ 워크북 결과 인용 |
| [10 Day-2 점검](10-day2-ops-checklist.md) | 정기 점검·업그레이드·복구 훈련·SLO 기록 | 플랫폼 운영자·SRE(사이트 신뢰성 엔지니어, Site Reliability Engineer) | 구축 후 상시 | 운영 기록·감사 추적, SLO는 문서 06 관측과 대조 |

## 사용법

1. 상황에 맞는 워크시트 파일을 엽니다 — 구축을 시작하는 단계면 `09-deployment-decision.md`, 이미 구축한 환경을 운영 중이면 `10-day2-ops-checklist.md`입니다.
2. 각 항목의 빈칸·체크박스를 자사 환경 값으로 채웁니다. 판단 근거가 필요한 칸은 연계 문서의 해당 절을 함께 확인합니다.
3. 채운 워크시트는 팀 내부에 보관하고, 환경이 바뀔 때마다 갱신합니다(특히 §10.4 SLO 기록과 §10.3 복구 훈련 결과).

## 중요(반드시 읽기)

- 이 워크시트는 **채워넣기 양식**입니다. 자동 계산은 없으며, 항목별로 직접 값을 적고 판단을 기록하는 용도입니다.
- GPU·노드·용량 등 **수치 산정이 필요하면** 사이징·비용 편(⑥)의 계산 워크북을 사용하세요. 본 워크시트는 그 결과를 옮겨 적는 칸만 제공합니다.
- 모든 항목은 작성 시점 기준이며 환경·릴리스별로 달라집니다. 적용 전 [상위 README](../README.md)와 공식 문서로 재확인하세요.
