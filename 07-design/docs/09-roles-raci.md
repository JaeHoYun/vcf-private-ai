# 09 — 역할과 책임 (RACI)

[← 목차로](../README.md)

Private AI 플랫폼을 만들고 운영하는 일은 한 팀이 다 하지 못합니다. 인프라·플랫폼·앱·보안·데이터가 단계마다 맞물리는데, "이 일은 누가 책임지고 누가 실행하나"가 흐릿하면 의사결정이 지연되고 책임이 빕니다. 이 문서는 AI 플랫폼 수명주기를 단계로 나눠, 단계마다 어느 역할이 무엇을 맡는지를 한 장의 RACI 표로 정리합니다.

> 이 표는 **출발점(템플릿)** 입니다. 조직마다 팀 경계와 이름이 다르므로, 자기 조직의 실제 팀에 매핑해 조정해 쓰십시오. 특정 기업의 조직도를 전제하지 않습니다.

---

## 9.1 역할과 RACI 표기

이 문서는 다섯 역할을 가정합니다. 실제 조직에서는 한 팀이 여러 역할을 겸하거나, 한 역할이 여러 팀에 나뉠 수 있습니다.

| 역할 | 맡는 범위 | 주로 맞닿는 시리즈 편 |
|------|-----------|----------------------|
| **인프라** | VCF·vSphere·네트워크·스토리지·GPU 하드웨어 | ① |
| **플랫폼** | PAIF·PAIS·VKS, 모델 서빙 플랫폼 | ①③·에이전트 가이드(플랫폼 측) |
| **앱** | RAG·에이전트 애플리케이션, 게이트웨이(BFF, 프런트 전용 백엔드) | ④·에이전트 가이드(앱 측) |
| **보안·거버넌스** | 보안 정책·격리·감사·컴플라이언스 | ⑤ |
| **데이터** | 데이터 준비·벡터DB 콘텐츠·문서 접근권한(ACL) | ②⑤(데이터 측) |

RACI는 각 단계에서 역할이 갖는 관여 방식을 네 글자로 표기합니다.

- **R(Responsible, 실행)** — 실제로 그 일을 수행한다.
- **A(Accountable, 책임)** — 결과에 대한 최종 책임·승인 권한을 가진다. 단계마다 **한 역할**에만 둔다.
- **C(Consulted, 협의)** — 진행 전 의견을 구한다(양방향).
- **I(Informed, 통보)** — 결과를 통보받는다(일방향).

## 9.2 수명주기 RACI

각 단계는 시리즈의 해당 편과 연결됩니다. 단계별로 **A는 하나**이며, A를 가진 역할이 실행(R)까지 겸하는 경우 `A·R`로 표기합니다.

| 단계 (시리즈 편) | 인프라 | 플랫폼 | 앱 | 보안·거버넌스 | 데이터 |
|------------------|:------:|:------:|:--:|:------------:|:------:|
| 요구사항·설계 (⑦·⑥) | C | A·R | C | C | C |
| 인프라·GPU 구축 (①) | A·R | C | I | C | I |
| 서빙 플랫폼 구축 (①③) | C | A·R | I | C | I |
| 데이터·벡터DB 구축 (②) | I | C | C | C | A·R |
| RAG·에이전트 앱 (④·에이전트 가이드) | I | C | A·R | C | C |
| 보안·격리·거버넌스 (⑤) | C | C | C | A·R | C |
| 운영·Day-2 (①·⑦) | R | A | C | C | C |
| 평가·가드레일 (④·에이전트 가이드의 평가 장) | I | C | R | A | C |

표를 가로로 읽으면 "이 단계는 누가 책임지고 누가 거드는가"가, 세로로 읽으면 "이 역할은 수명주기 전체에서 언제 전면에 서는가"가 보입니다. 예컨대 보안·거버넌스는 대부분 단계에서 C로 관여하다가 보안·격리·거버넌스 단계와 평가·가드레일에서 A로 전면에 섭니다.

## 9.3 사용법·주의

- **A는 단계마다 하나로 유지합니다.** A가 둘이면 책임이 갈리고, 없으면 의사결정이 멈춥니다. 조직에 맞게 옮기되 이 규율은 지키십시오.
- **역할명을 자기 조직 팀에 매핑하십시오.** "플랫폼 팀"이 없다면 그 책임이 인프라와 앱 중 어디로 가는지를 먼저 정합니다.
- **단계 경계는 시리즈 편과 느슨하게 대응**하므로, 상세 활동은 각 편으로 들어가 확인합니다(예: 격리 통제는 [⑤ 보안·거버넌스](../../05-security/README.md), 사이징은 [⑥ 사이징·용량·비용](../../06-sizing-cost/README.md)).
- 이 표는 책임의 밑그림이며, 실제 운영에서는 변경 관리·온콜·승인 흐름 같은 세부 절차로 보강해야 합니다.
